## 是什么

根据研究主题自动生成**可视化 HTML 报告**，生成后直接发送给用户。

支持：多风格切换 · 来源自动标注 · 直接发送文件（无需任何部署环境）

---

## 快速开始

### 基本语法

```
报告 [主题]，[风格]
```

**例子：**
```
报告 Meta 广告算法新规，infographic
报告 DTC 品牌出海，dashboard + magazine
报告 AI 工具选型，极简白
```

## 四种输入模式

### 模式 A｜指定风格
直接说风格名称，精确匹配。

```
帮我写一份关于 [主题] 的报告，风格是 [风格名]
```

### 模式 B｜让 AI 推荐
不确定用什么风格，让 AI 根据主题判断。

```
帮我研究 [主题]，你推荐最适合的风格
```

AI 会根据主题特点推荐：
- 算法/规则解读 → **infographic**（流程图+颜色编码）
- 高管汇报 → **dashboard**（KPI 卡片）
- 学术/内容营销 → **magazine**（衬线引文）

### 模式 C｜描述感觉
不给风格名，给感觉描述。

```
"要一个看起来很权威的、像杂志封面的报告"
"科技感强一点，暗色系"
"活泼一点，多用 emoji"
```

### 模式 D｜一次生成多个风格
一个主题，N 个版本同时出。

```
生成 5 个版本的报告：信息图、仪表盘、暗黑科技、极简、复古
```

## 10种风格

| 风格 | 关键词 | 适用场景 |
|------|--------|---------|
| `dashboard` | 仪表盘 | 高管汇报、数据呈现 |
| `infographic` | 信息图 | **算法/政策解读（默认推荐）** |
| `magazine` | 杂志社论 | 学术、权威感 |
| `dark` | 暗黑科技 | 程序员/极客受众 |
| `minimal` | 极简白 | 简约北欧风 |
| `retro` | 复古报纸 | 老式排版质感 |
| `gradient` | 渐变现代 | 现代柔和暖色 |
| `data` | 数据优先 | 表格为主，严肃专业 |
| `playful` | 活力创意 | emoji 点缀，轻松活泼 |
| `professional` | 专业蓝 | 企业信任感 |

## 质量标准

### 必须标注来源

每个 HTML 底部必须有参考文献区，每个模块标题旁必须有 `[来源]` 标签：

```html
<div class="sources">
  <h4>📚 参考来源（点击可查看原文）</h4>
  <ul>
    <li><a href="https://官方URL" target="_blank">来源名称</a></li>
  </ul>
</div>

<div class="section-title">
  模块标题
  <a href="[来源URL]" target="_blank" class="src-tag">[来源]</a>
</div>
```

### 来源优先级

1. **官方文档**（Meta透明度中心、官方博客）
2. **权威媒体**（TechCrunch、Social Media Today）
3. **行业分析**（Anchour、GrowthHQ）
4. **案例数据**（标注"样本量N="或"来源：[平台]内部数据"）

### 自检清单

每个报告生成后必须通过：

- [ ] `<!DOCTYPE>` / `<html>` / `<head>` / `<body>` 完整
- [ ] 所有标签正确闭合，无残标
- [ ] 有真正的可视化元素（进度条/图表/颜色编码）
- [ ] 不是纯文字堆砌
- [ ] 每个模块有来源引用
- [ ] 页面能正常渲染（截图确认）

> 宁可重做一个精品，不发十个垃圾。

## 交付方式

### 默认：直接发送 HTML 文件

**生成报告后，直接用 message 工具发送文件给用户。**

用户收到后浏览器打开 `.html` 即可，**完全离线、立即可用**，无需注册任何服务。

```
message(
  action="send",
  channel="[当前渠道，如 telegram/discord]",
  filePath="/路径/to/report.html"
)
```

### 可选：自己部署到 Vercel

如需自己部署获得永久在线链接：

**Step 1：** 注册 Vercel → https://vercel.com（GitHub 账号免费注册）

**Step 2：** 本地安装 CLI 并部署
```bash
npm i -g vercel
cd [报告目录]
vercel login
vercel deploy --prod
```

多版本项目结构：
```
├── index.html              # 入口索引页
├── v01-infographic.html
├── v02-dashboard.html
└── ...
```

## 示例对话

### 示例 1：单主题单风格

**用户：**
> 帮我写一份关于 Meta 广告算法 2026 新规的报告，用信息图风格

**AI 响应流程：**
1. 搜索/整理 Meta 官方最新政策文献
2. 生成 infographic 风格 HTML（进度条+颜色编码+流程图）
3. 每个模块添加 `[来源]` 标签，底部加完整可点击来源列表
4. 自检（结构 + 可视化 + 来源标注）
5. **直接发送 HTML 文件给用户**

---

### 示例 2：单主题多风格

**用户：**
> 帮我生成 3 个版本的报告：dashboard、暗黑科技、杂志风

**AI：**
> 收到，3 个版本同时生成，预计 10 分钟完成

**交付物：**
- `v01-dashboard.html` — KPI 卡片，蓝色主调
- `v02-dark.html` — 暗黑终端风，绿色代码字体
- `v03-magazine.html` — 衬线引文，学术感

---

### 示例 3：描述感觉

**用户：**
> 要一个看起来很权威的、像杂志封面的报告

**AI：**
> 理解：权威感 + 杂志风格 → 推荐 `magazine` 或 `professional-blue`
