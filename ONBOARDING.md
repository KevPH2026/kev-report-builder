# Report Builder Skill — Onboarding Guide

## 这是什么

一个 AI 驱动的可视化报告生成器。根据研究主题 + 风格偏好，自动生成可部署上线的 HTML 报告。

---

## 快速上手（3步）

### Step 1：告诉我你要研究什么

```
研究主题 + 风格偏好
```

**示例：**
- "Meta 广告算法新规，用信息图风格"
- "DTC 品牌出海，3个版本的报告：dashboard、暗黑科技、杂志风"
- "一份关于 AI 工具选型的极简白报告"

---

## 两种工作模式

### 模式 A：用户选风格

直接告诉我要什么风格，我会生成对应的可视化报告。

```
风格列表：dashboard / infographic / magazine / dark / minimal / retro / gradient / data / playful / professional
```

### 模式 B：让 AI 推荐风格

```
"帮我研究 [主题]，你推荐最适合的风格"
```

我会根据主题特点推荐最合适的风格组合：
- 算法/规则解读 → **infographic**（流程图+颜色编码）
- 高管汇报 → **dashboard**（KPI 卡片）
- 学术/内容营销 → **magazine**（衬线引文）

### 模式 C：用户描述想要的感觉

```
"要一个看起来很权威的、像杂志封面的报告"
"科技感强一点，暗色系"
"活泼一点，多用 emoji"
```

我会理解你的描述，匹配合适的风格。

### 模式 D：一次生成多个风格

```
"帮我生成 5 个版本的报告，覆盖：信息图、仪表盘、暗黑科技、极简、复古"
```

一次生成多个版本，入口索引页会自动展示所有版本。

---

## 来源引用说明

所有报告都遵循「每个结论有来源，每个模块可查证」原则：

- 每个 HTML 底部有完整的可点击参考文献列表
- 每个模块标题旁有 `[来源]` 标签，点击跳转到原始文档
- 区分「Meta 官方声明」和「第三方推测」，不混为一谈

---

## 输出形式

| 内容 | 说明 |
|------|------|
| 在线链接 | Vercel 托管，可直接分享 |
| 文件位置 | 本地备份路径 |
| 版本列表 | 多版本时提供所有链接 |

---

## 质量承诺

每个报告生成后必须经过自检：
- ✅ 标签完整闭合
- ✅ 有可视化元素（非纯文字）
- ✅ 每个模块有来源引用
- ✅ 页面能正常渲染

如果自检不通过，会重新生成，不会发出垃圾版本。
